## [Oak：专为 AI 智能体工作流设计的新版本控制系统](https://oak.space/oak/oak) ⭐️ 8.0/10

一位开发者发布了 Oak，这是一款专为 AI 智能体工作流设计的早期开源版本控制系统，其虚拟挂载功能允许智能体无需下载完整代码库即可开展工作。Oak 已在自身平台上自托管运行数月且未使用 Git 备份，但目前尚不支持 Windows，也缺少 CI、问题追踪和评论等功能。 随着 AI 编程智能体日益普及，软件开发的底层工具可能需要重新设计，以适应与人类开发者习惯截然不同的并行自动化工作流。Oak 是首批尝试围绕智能体需求原生构建版本控制基础设施的项目之一，而非对 Git 等现有工具进行改造。 Oak 的核心差异化功能是虚拟挂载，允许智能体仅访问代码库中所需的部分，无需克隆整个仓库，从而在无需处理 Git worktree 复杂性的情况下并行运行多个任务。该项目目前仍缺少许多主要功能，也没有 Windows 版本，暂不适合在生产环境中使用。

hackernews · zdgeier · 6月22日 15:37 · [社区讨论](https://news.ycombinator.com/item?id=48631726)

**背景**: Git 是全球开发者广泛使用的主流版本控制系统，用于追踪代码变更、协作开发和管理项目历史。Git worktree 是 Git 的现有功能，允许在不同目录中同时检出多个分支，但管理起来较为繁琐，尤其是在多个自动化智能体并行运行时。AI 编程智能体——能够自主编写、编辑和提交代码的软件——在软件开发中的应用日益广泛，对相关工具提出了新的需求。Oak 所采用的虚拟挂载技术允许进程像访问本地文件系统一样访问远程或部分文件系统，无需完整下载。

## [Moebius 0.2B 图像修复模型通过 WebGPU 移植至浏览器运行](https://simonwillison.net/2026/Jun/22/porting-moebius/#atom-everything) ⭐️ 7.0/10

Simon Willison 借助 Claude Code 作为 AI 编程助手，将原本需要 PyTorch 和 NVIDIA CUDA 才能运行的 Moebius 0.2B 图像修复模型，成功移植到浏览器中，通过 WebGPU 和 ONNX Runtime Web 实现完全本地运行，并发布了可在线体验的演示页面。 这个项目表明，小型但性能出色的 AI 模型可以直接在浏览器中运行，无需服务器端基础设施或专用硬件，大幅降低了向用户部署 AI 图像编辑工具的门槛。 该移植方案采用基于 WebGPU 后端的 ONNX Runtime Web，比 Transformers.js 更底层；整个开发过程作为副项目完成，利用的是等待另一个编程智能体（Codex Desktop）处理其他任务时的空闲时间。

rss · Simon Willison · 6月22日 23:43

**背景**: 图像修复（Image Inpainting）是一种让模型用合理内容填充图像中被遮罩或缺失区域的技术，常用于去除物体和修复照片。Moebius 是一个新发布的深度学习模型，仅有 2 亿个参数，但声称性能可媲美规模大得多的百亿参数模型。WebGPU 是一种现代浏览器 API，可直接低开销地访问设备 GPU，用于渲染和通用计算，从而使机器学习推理能够在浏览器标签页中于客户端运行。ONNX Runtime Web 是一个跨平台推理引擎，可在浏览器内执行 ONNX 格式的模型，并可选择通过 WebGPU 加速。

## [印度 BharatGen 加入 AI 联盟 Project Tapestry 联邦前沿 AI 项目](https://www.reddit.com/r/artificial/comments/1uckw98/indias_bharatgen_commits_to_anchor_indias_role_in/) ⭐️ 6.0/10

印度政府支持的 AI 项目 BharatGen 正式承诺，将作为印度在 AI 联盟 Project Tapestry 中的核心参与方。Project Tapestry 是一个通过全球联邦协作开发前沿 AI 模型的开源平台，此次公告恰逢 G7 国家将 AI 主权列为核心政策议题。 Project Tapestry 是在中心化实验室模式之外构建具有竞争力的前沿 AI 模型的重要尝试，允许参与国保留对数据、模型和部署的主权控制。BharatGen 的加入表明，主要发展中经济体正在积极寻求替代西方或中国 AI 基础设施依赖的路径。 Project Tapestry 的架构设计允许多个国家和机构联合训练前沿模型，同时各方保留本地控制权和长期独立性——这种联邦方式避免了敏感数据的集中存储。然而，在前沿规模上进行联邦开发在很大程度上仍未经验证，面临算力共享、数据治理以及多方协调模型发布决策等挑战。

## [Gray Swan 创始人解释为何 AI 安全不同于传统网络安全](https://www.latent.space/p/gray-swan) ⭐️ 7.5/10

OpenAI 董事会成员 Zico Kolter 与 Gray Swan 首席执行官 Matt Fredrikson 在 Latent Space 播客上发表观点，认为 AI 安全是一门与传统网络安全本质上不同的学科。对话涵盖了红队测试方法、AI 安全概念以及 AI 系统所面临的独特威胁环境。 随着 AI 系统深度融入企业产品和工作流程，理解其独特漏洞——如提示词注入、越狱攻击和幻觉问题——对开发者和安全团队至关重要。Gray Swan 已与 OpenAI、Anthropic 等主要 AI 实验室合作，代表着一个专注于 AI 安全的新兴专业领域。 Gray Swan 由卡内基梅隆大学的计算机科学家创立，提供覆盖 AI 模型和智能体完整生命周期的安全套件——从数据整理到实时监控——包括针对越狱攻击、提示词注入以及具有工具访问权限的自主 AI 智能体威胁的防御措施。该公司平台还支持 MCP 和工具集成，保护连接到数据库、API 和文件的 AI 系统。

rss · Latent Space · 6月22日 21:06

**背景**: 红队测试是一种安全实践，由专门团队尝试攻击或利用系统漏洞，以便在恶意行为者发现之前找到安全隐患。在传统网络安全中，这涉及探测防火墙、访问控制和软件漏洞。对于 AI 系统，红队测试具有新的维度：测试人员必须探测提示词注入（通过精心设计的输入操控 AI）、越狱攻击（绕过安全护栏）、数据泄露和有害输出等行为——这些威胁在传统 IT 安全中没有直接对应物。Zico Kolter 是卡内基梅隆大学教授及 OpenAI 董事会成员，Matt Fredrikson 是卡内基梅隆大学研究员及 Gray Swan 联合创始人。
